# 联机锄地功能使用教程

## 一、功能概述

联机锄地功能允许最多4名玩家组队自动执行锄地路线，实现自动组队、同步等待、协调战斗、多世界轮换等功能。系统通过协调服务器（Coordinator Server）实现玩家间的状态同步和协调。

联机锄地基于莫酱（Mno）的锄地一条龙JS，所有资源和代码都需要JS支持，所以必须订阅锄地一条龙JS

目前在服务器内测中，有兴趣可申请，服务器源码后续开源，可自行搭建服务器。

注意：独立任务中新增“锄地一条龙”任务，联机锄地包含在其，配置组可以添加独立任务中的锄地一条龙，右击配置独立任务，启用联机锄地，即可在配置组中使用联机锄地功能。

## 二、配置设置详解

### 1. 基础联机配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **启用联机模式** | `false` | 必须开启才能使用联机功能 |
| **联机角色** | `host` | `host`=房主，`member`=成员 |
| **协调服务器地址** | `https://bgi-sync.example.com` | 需要替换为实际地址 | 
| **玩家名称** | 空 | 玩家的游戏昵称（用于识别加入世界） |
| **玩家UID** | 空 | 游戏内UID，用于进入世界和多世界切换 |

### 2. 房主配置（host）

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **房间白名单** | 空 | 逗号分隔的玩家名称，限制可加入的玩家 | 支持备注名识别 | 注意需要把房主自己也要加到白名单中 |
| **期望玩家数量** | `4` | 房间期望人数（2-4），用于判断人齐条件 |
| **集合点等待超时** | `60` | 同步点等待超时（秒） |
| **最低开始人数** | `0` | 0为自动识别房间人数 |
| **战斗超时时间** | `120` | 联机模式战斗超时（秒），同步覆盖成员配置 |
| **战斗额外等待时间** | `60` | 同步点超时后为Fighting成员额外等待（秒） |
| **重新加入最大等待** | `300` | 同步点超时后为Rejoining/Reviving成员额外等待（秒） |
| **传送点必同步** | `false` | 启用后所有传送点都作为同步等待点 |
| **万叶玩家序号** | `0` | 0=不指定，1-4=对应玩家序号 | 功能未完善，暂有BUG |

### 3. 成员配置（member）

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **成员加入方式** | `random` | `random`=随机加入现有房间，`byHostName`=指定房主名称 |
| **目标房主名称** | 空 | 成员加入时指定的房主玩家名称（byHostName模式） |
| **组队等待超时** | `600` | 组队等待超时（秒），超时后停止联机锄地 |
| **组队超时动作** | `0` | 0=结束任务，1=现有人数锄地 |

### 4. 多世界连续锄地配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **启用多世界连续锄地** | `false` | 房主设定，完成一个世界后轮换到下一个玩家的世界 |
| **多世界锄地轮数** | `2` | 多世界锄地轮数（1-4），按加入顺序依次成为房主 |

## 三、使用流程

### 1. 准备工作

1. **确保游戏已启动**并登录到游戏主界面
2. **配置协调服务器地址**（需要可访问的协调服务器）
3. **设置玩家名称和UID**（必须准确）
4. **选择联机角色**：房主或成员

### 2. 房主操作流程

#### 步骤1：创建房间
- 设置`联机角色`为`host`
- 配置房间参数（白名单、期望人数、超时时间等）
- 启动任务，系统会自动：
  - 连接协调服务器
  - 创建房间并生成房间码
  - 等待成员加入

#### 步骤2：等待成员加入
- 系统自动处理成员加入申请
- 白名单模式下只允许指定玩家加入
- 达到期望人数或超时后自动开始

#### 步骤3：执行联机锄地
- 系统自动：
  - 同步路线列表给所有成员
  - 在同步点等待所有玩家到达
  - 协调战斗开始和结束
  - 处理异常情况（玩家掉线、战斗失败等）

#### 步骤4：多世界轮换（如启用）
- 完成当前世界后自动切换到下一个玩家的世界
- 新玩家成为房主，继续执行路线

### 3. 成员操作流程

#### 步骤1：加入房间
- 设置`联机角色`为`member`
- 选择加入方式：
  - `random`：随机加入有空位的房间
  - `byHostName`：加入指定房主的房间
- 启动任务，系统会自动：
  - 连接协调服务器
  - 查找并加入房间
  - 进入房主世界

#### 步骤2：同步配置
- 自动同步房主的配置（超时时间、战斗设置等）
- 下载房主选择的路线列表

#### 步骤3：跟随执行
- 在同步点等待房主指令
- 自动战斗和拾取
- 异常时自动尝试重新加入

## 四、核心功能详解

### 1. 同步点机制
- **战斗前同步点**：每个战斗点前等待所有玩家到达
- **传送点同步**（可选）：所有传送点都作为同步点
- **超时处理**：超时后根据成员状态决定额外等待或跳过

### 2. 状态监控与恢复
- **世界状态监控**：持续检测是否仍在联机世界
- **心跳机制**：5秒一次心跳，30秒超时判定掉线
- **恢复窗口**：检测到异常后30秒恢复窗口，允许瞬态干扰恢复
- **协调停止**：异常时通知所有玩家协调停止，避免失同步

### 3. 异常处理
- **玩家掉线**：自动检测并处理，剩余玩家继续或停止
- **战斗失败**：根据配置重试或跳过
- **网络中断**：尝试重连，失败后协调停止
- **游戏崩溃**：检测到异常后停止所有玩家任务

## 五、常见问题与解决方案

### 1. 无法连接协调服务器
- 检查网络连接
- 确认服务器地址正确
- 检查防火墙设置

### 2. 成员无法加入房间
- 确认房主已创建房间
- 检查白名单设置
- 确认房间未满员（最多4人）

### 3. 同步点等待超时
- 检查所有玩家游戏是否正常运行
- 调整`集合点等待超时`设置
- 检查网络延迟

### 4. 战斗不同步
- 确认`战斗超时时间`设置合理
- 检查玩家电脑性能差异
- 考虑增加`战斗额外等待时间`

### 5. 多世界切换失败
- 确认所有玩家UID正确
- 检查每个玩家都有权限进入对方世界
- 调整`多世界锄地轮数`设置

## 六、最佳实践建议

1. **网络环境**：确保所有玩家网络稳定，延迟低
2. **游戏设置**：统一游戏分辨率和图形设置（推荐1920x1080）
3. **队伍配置**：提前协调队伍角色和技能搭配
4. **路线选择**：房主选择适合联机的路线（避免狭窄地形、复杂路线）
5. **沟通协调**：建议使用语音沟通，特别是异常情况处理
6. **测试运行**：首次联机前先进行单人测试，确保功能正常
7. **备份配置**：重要配置变更前备份配置文件

## 七、高级配置技巧

### 1. 性能优化
- 调整`识别间隔`减少CPU占用
- 设置`不运行时段`避开高峰期
- 使用`仅指定怪物模式`提高效率

### 2. 容错配置
- 增加`重新加入最大等待时间`应对网络波动
- 设置`最大连续跳过路线次数`防止无限循环
- 启用`传送点必同步`提高同步精度

### 3. 调试技巧
- 设置`最低开始人数=1`进行单人调试
- 启用`调试模式`跳过路线一致性验证
- 使用`固定调试线路`测试特定路线

## 八、安全注意事项

1. **账号安全**：不要分享游戏账号信息
2. **白名单设置**：建议使用白名单限制可加入玩家
3. **服务器安全**：使用可信的协调服务器
4. **数据隐私**：注意玩家名称和UID的隐私保护
5. **游戏规则**：遵守游戏服务条款，合理使用自动化功能
